# 虚拟歌手中文曲排行榜 - 特定视频抓取&计分工具
该工具用于抓取B站指定BV号视频的播放/互动数据，并基于预设规则计算视频的周刊得分、生成排行榜，适配虚拟歌手中文曲排行榜的计分逻辑。

## 核心功能
1. **多格式BV号导入**：支持手动输入/从TXT/CSV/Excel文件批量读取BV号（自动去重、格式校验）
2. **批量数据抓取**：多线程抓取视频数据，带重试机制和请求间隔控制，避免请求过载
3. **增量计分排行**：对比两期抓取数据，计算各项数据增量，按规则计算最终得分并生成排行榜
4. **灵活使用方式**：支持命令行模式和交互式菜单模式

## 环境准备
### 依赖安装
工具基于Python开发，需安装以下依赖包：
```bash
pip install requests pandas tqdm openpyxl
```
- `requests`：发送HTTP请求抓取视频数据
- `pandas`：数据处理与CSV/Excel文件读写
- `tqdm`：显示抓取进度条
- `openpyxl`：处理Excel文件（.xlsx/.xls），可选（仅导入Excel时需要）

## 使用方法
### 1. 基础使用（交互式菜单）
直接运行脚本，进入交互式菜单操作：
```bash
python 1.1虚拟歌手中文曲排行榜 特定视频抓取计分工具.py
```
菜单选项说明：
| 序号 | 功能 | 操作说明 |
|------|------|----------|
| 1 | 抓取指定BV号视频数据 | 选择输入模式（手动输入/文件导入），完成后自动生成当日日期命名的CSV数据文件 |
| 2 | 计算增量&生成排行榜 | 输入上期、本期数据CSV文件路径，自动计算增量和得分，生成排行榜CSV |
| 3 | 自动处理最新两期数据 | 自动识别目录下最新的两个`YYYYMMDD_specified_video_data.csv`文件，一键计算排行 |
| 4 | 退出 | 退出程序 |

### 2. 命令行模式（批量/脚本化）
支持直接通过命令行参数指定操作，适合自动化脚本调用：

#### 模式1：抓取指定BV号数据
```bash
# 单个BV号
python 脚本名.py --mode fetch --bvids BV1LXwyzkEqo

# 多个BV号
python 脚本名.py --mode fetch --bvids BV1LXwyzkEqo BV12345678901
```

#### 模式2：计算两期数据排行
```bash
python 脚本名.py --mode rank --last 20240101_specified_video_data.csv --current 20240102_specified_video_data.csv
```

#### 模式3：自动处理最新两期数据
```bash
python 脚本名.py --mode auto
```

## 数据文件说明
### 1. 抓取数据文件（YYYYMMDD_specified_video_data.csv）
抓取生成的原始数据文件，包含字段：
| 字段 | 说明 |
|------|------|
| bvid | 视频BV号 |
| title | 视频标题 |
| up主 | 视频UP主名称 |
| 播放量/弹幕数/评论数/收藏数/投币数/点赞数 | 视频对应数据 |
| 抓取时间 | 数据抓取的时间戳 |

### 2. 排行榜结果文件（specified_rank_result.csv）
计分排行生成的结果文件，核心字段：
| 字段 | 说明 |
|------|------|
| 排名 | 按最终得分降序的排名 |
| bvid | 视频BV号 |
| title_本期 | 本期视频标题 |
| up主_本期 | 本期UP主名称 |
| 最终得分 | 按规则计算的总得分 |
| 播放增量/弹幕增量/评论增量/收藏增量/投币增量 | 两期数据的差值（非负） |
| 点赞有效增量 | 点赞增量上限为投币增量的2倍（自定义规则） |

## 计分规则说明
最终得分 = 播放得分 + 互动得分 + 收藏得分 + 投币得分 + 点赞得分
- 播放得分 = 播放增量 × D（默认D=1）
- 互动得分 = (评论增量 + 弹幕增量) × A × 15（默认A=1）
- 收藏得分 = 收藏增量 × B（默认B=1）
- 投币得分 = 投币增量 × C（默认C=1）
- 点赞得分 = 点赞有效增量（点赞增量 ≤ 投币增量×2）

可在`calculate_rank_and_score`函数中修改A/B/C/D参数调整计分权重。

## 注意事项
1. **BV号格式**：仅支持标准B站BV号（12位，以BV开头），工具会自动校验格式，支持从B站视频URL中提取BV号
2. **文件编码**：读取文件时优先使用UTF-8，TXT文件若读取失败会自动尝试GBK编码；生成的CSV文件为UTF-8-SIG编码（兼容Excel）
3. **请求限制**：默认请求间隔2秒，最大重试3次，多线程最大4个工作线程，可在`Config`类中调整：
   - `REQUEST_INTERVAL`：请求间隔（秒）
   - `RETRY_TIMES`：重试次数
   - `MAX_WORKERS`：最大线程数
4. **数据准确性**：B站API返回的播放量等数据为平台展示值，可能存在延迟或四舍五入
5. **异常处理**：抓取失败的视频会在日志中提示，最终仅保存成功抓取的数据

## 常见问题
1. **Excel文件读取失败**：需安装openpyxl（`pip install openpyxl`），且确保Excel文件路径无中文/特殊字符
2. **BV号导入后无有效数据**：检查文件中BV号格式是否正确，或是否存在重复/无效BV号
3. **排行榜计算提示无匹配BV号**：确保两期数据CSV文件中有相同的BV号（仅对交集数据计算）
4. **请求被拒绝**：B站API有访问频率限制，可增大`REQUEST_INTERVAL`或减少`MAX_WORKERS`时选择“UTF-8编码”，或用记事本打开后另存为ANSI编码
4. **请求超时/重试多次失败**：检查网络，或增大`REQUEST_INTERVAL`/`RETRY_TIMES`
